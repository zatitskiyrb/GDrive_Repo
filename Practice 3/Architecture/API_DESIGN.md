# API Design

**Product:** Dice Randomizer
**API Style:** Отсутствует
**Last Updated:** 2026-04-30

---

## Внешний API

**API нет.** Приложение полностью клиентское — никаких HTTP-запросов к серверу не выполняется.

---

## Внутренний JavaScript API (модули)

Публичный интерфейс модуля `app.js`:

### `rollDice()`

Запускает бросок: генерирует числа, запускает анимацию, воспроизводит звук, обновляет DOM.

```
Вход:  нет
Выход: нет (side effects: обновление DOM)
Вызов: по клику на кнопку "Бросить кубики"
```

### `getRandomDie()`

Возвращает случайное целое число от 1 до 6.

```
Вход:  нет
Выход: number (1–6)
```

### `renderDie(element, value)`

Обновляет DOM-элемент кубика: отображает точки для заданного значения.

```
Вход:  element — HTMLElement (DOM-узел кубика)
       value   — number (1–6)
Выход: нет (side effects: обновление DOM)
```

### `playSound()`

Воспроизводит звуковой эффект броска.

```
Вход:  нет
Выход: нет (может тихо упасть, если браузер блокирует автовоспроизведение)
```

---

## Примечание

Если в будущих версиях появится история бросков или мультиплеер, потребуется REST API. На данном этапе — не нужен.
