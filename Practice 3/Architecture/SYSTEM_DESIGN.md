# System Design

**Product:** Dice Randomizer
**Architecture Pattern:** Single-Page Static Application
**Quality Mode:** MVP
**Last Updated:** 2026-04-30

---

## Общая архитектура

Приложение — полностью клиентское. Никакого сервера приложений нет.
Nginx выступает исключительно как файловый сервер (раздаёт HTML/CSS/JS/assets).

```
Browser
  └── index.html
        ├── style.css        (внешний вид, анимации)
        ├── app.js           (логика броска, управление DOM)
        └── assets/
              └── roll.mp3   (звуковой эффект)
```

---

## Слои

| Слой | Что делает | Файлы |
|---|---|---|
| **Presentation** | HTML-разметка, структура страницы | `index.html` |
| **Style** | Мультяшный дизайн, CSS-анимации | `style.css` |
| **Logic** | Генерация чисел, управление состоянием, DOM | `app.js` |
| **Assets** | Статические ресурсы (звук) | `assets/` |

## Компоненты приложения

### 1. Dice Component (визуальный кубик)
- Отображает точки (dots) для значений 1–6
- Принимает CSS-класс анимации при броске
- Два экземпляра на странице

### 2. Roll Button
- Запускает бросок
- Блокируется на время анимации (атрибут `disabled`)
- Разблокируется по завершении

### 3. Sum Display
- Отображает "Сумма: X"
- Скрыт до первого броска
- Обновляется после каждого броска

### 4. Sound Manager
- Воспроизводит `roll.mp3` при каждом броске
- Обрабатывает отказ браузера воспроизвести звук (no crash)

## Поток данных

```
Пользователь нажимает кнопку
        ↓
Кнопка блокируется (disabled = true)
        ↓
Запускается CSS-анимация на обоих кубиках
Воспроизводится звук
        ↓
Генерируются два случайных числа (1–6)
        ↓
setTimeout ~1000ms (ждём конца анимации)
        ↓
Числа рендерятся в DOM (точки кубиков)
Сумма вычисляется и отображается
        ↓
Анимация снимается
Кнопка разблокируется (disabled = false)
```

## Решения по дизайну

| Решение | Альтернатива | Почему выбрано |
|---|---|---|
| Vanilla JS | React / Vue | Нет смысла в фреймворке для 50 строк логики |
| CSS-анимация | JS-анимация (canvas) | Проще, нативная поддержка, плавнее |
| Точки через DOM | SVG / Canvas | Читаемо, легко кастомизировать через CSS |
| Nginx alpine | Node.js сервер | Нет динамики — nginx достаточен и легче |
