## Use Case: Dice Rolling

### Scenario A: Успешный бросок кубиков

**Preconditions:**
- Приложение открыто в браузере
- Кнопка "Бросить кубики" активна

**Steps:**
1. Пользователь нажимает кнопку "Бросить кубики"
2. Оба кубика начинают анимацию броска
3. Анимация завершается через ~1 секунду
4. На каждом кубике отображается случайное число (1–6)
5. Под кубиками отображается сумма: "Сумма: X"

**Expected Outcome:**
Два кубика показывают разные или одинаковые значения (1–6), сумма вычислена корректно (2–12).

**Edge Cases:**
- **Одинаковые значения:** оба кубика показывают одно число — это нормально, система не перебрасывает
- **Минимальная сумма (1+1=2):** корректно отображается "Сумма: 2"
- **Максимальная сумма (6+6=12):** корректно отображается "Сумма: 12"

---

### Scenario B: Повторный бросок

**Preconditions:**
- Уже был совершён минимум один бросок
- Предыдущие значения отображены на экране

**Steps:**
1. Пользователь снова нажимает кнопку "Бросить кубики"
2. Предыдущие значения исчезают / сбрасываются
3. Анимация броска воспроизводится заново
4. Новые случайные значения отображаются на кубиках
5. Сумма обновляется

**Expected Outcome:**
Новые значения заменяют предыдущие, сумма пересчитана.

**Edge Cases:**
- **Случайно то же значение:** допустимо, не баг

## Use Case: Animation & Sound

### Scenario A: Анимация броска воспроизводится

**Preconditions:**
- Пользователь нажал кнопку "Бросить кубики"

**Steps:**
1. Кнопка блокируется (disabled) немедленно
2. Оба кубика начинают анимацию (вращение / тряска)
3. Звуковой эффект броска воспроизводится
4. Через ~1 секунду анимация останавливается
5. Финальные значения зафиксированы
6. Кнопка снова становится активной

**Expected Outcome:**
Анимация плавная, звук воспроизводится один раз за бросок, кнопка разблокируется после завершения.

**Edge Cases:**
- **Звук заблокирован браузером:** анимация всё равно работает, звук тихо пропускается (no crash)
- **Медленное устройство:** анимация может быть короче, но значения всё равно отображаются корректно

---

### Scenario B: Кнопка заблокирована во время броска

**Preconditions:**
- Анимация броска уже идёт

**Steps:**
1. Пользователь пытается нажать кнопку повторно во время анимации
2. Клик игнорируется (кнопка disabled)

**Expected Outcome:**
Двойной бросок не происходит, анимация не сбрасывается.

## Use Case: UI Interaction

### Scenario A: Первый запуск приложения

**Preconditions:**
- Пользователь открыл URL приложения в браузере

**Steps:**
1. Страница загружается
2. Отображаются два кубика в начальном состоянии (пустые или с "?")
3. Видна кнопка "Бросить кубики"
4. Сумма не отображается (или скрыта) до первого броска

**Expected Outcome:**
Интерфейс понятен сразу: два кубика + кнопка. Ребёнок без инструкций понимает, что нужно нажать.

**Edge Cases:**
- **Узкий экран (мобильный):** интерфейс адаптируется, ничего не обрезается

## Use Case: Deployment

### Scenario A: Запуск через Docker

**Preconditions:**
- Docker установлен на машине
- Пользователь в директории проекта

**Steps:**
1. Пользователь выполняет `docker build -t dice-randomizer .`
2. Docker собирает образ
3. Пользователь выполняет `docker run -p 8080:80 dice-randomizer`
4. Пользователь открывает `http://localhost:8080` в браузере
5. Приложение загружается и работает

**Expected Outcome:**
Приложение доступно в браузере без дополнительной настройки.

**Edge Cases:**
- **Порт 8080 занят:** пользователь может использовать другой порт (`-p 3000:80`)
